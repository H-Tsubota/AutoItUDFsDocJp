Function Reference

_FTP_FilePut

FTPサーバー上にファイルを置きます。

#Include <FTPEx.au3>
_FTP_FilePut($l_FTPSession, $s_LocalFile, $s_RemoteFile [, $l_Flags = 0 [, $l_Context = 0]])

 

パラメータ

$l_FTPSession _FTP_Connect()の返し値
$s_LocalFile ローカルファイル
$s_RemoteFile ファイルのリモート位置
$l_Flags [オプション] 転送時の環境設定を指定します。アプリケーションはいずれかの転送タイプを選択する必要があり、ファイルをどのようにキャッシングするかを表すいずれかのフラグを指定します。

転送タイプは次の値のいずれか一つ:
$FTP_TRANSFER_TYPE_ASCII - FTPのASCII転送メソッド(タイプA)を使用してファイルを転送します。制御、フォーマット情報はローカルと同じものに変換されます。
$FTP_TRANSFER_TYPE_BINARY - FTPのイメージ転送メソッド(タイプI)を使用してファイルを転送します。ファイルは変化せず完全に同一なまま転送されます。デフォルトの転送メソッドです。
$FTP_TRANSFER_TYPE_UNKNOWN - デフォルトは$FTP_TRANSFER_TYPE_BINARY
$INTERNET_FLAG_TRANSFER_ASCII - ASCIIとしてファイルを転送します。
$INTERNET_FLAG_TRANSFER_BINARY - バイナリとしてファイルを転送します。

以下はファイルのキャッシュ方法を制御するために使用されます。アプリケーションは1つまたは複数の値を使用できます:
$INTERNET_FLAG_HYPERLINK - ネットワークからアイテムをリロードするかどうかを決定する際、 有効期限、最終変更時刻がサーバーから返されなければ強制的にリロードします。
$INTERNET_FLAG_NEED_FILE - ファイルをキャッシュできない場合、一時ファイルを作成します。
$INTERNET_FLAG_RELOAD - リクエストされたファイル、オブジェクト、ディレクトリリストをキャッシュではなく元サーバーから強制的にダウンロードします。
$INTERNET_FLAG_RESYNCHRONIZE - 最後にダウンロードした時際のリソースから変更があった場合、HTTPソースをリロードします。全てのFTPリソース、Gopherリソースががリロードされます。
$l_Context [オプション] このサーチと任意のアプリケーションデータを関連付けるアプリケーション定義の値を保持した変数。
ステータスコールバック関数を設定するためにアプリケーションが既に_FTP_SetStatusCallback()を呼び終わっている場合のみ使用できます。

 

返し値

成功: 1
失敗: 0を返し、@errorを設定します。

 

注意

$s_LocalFile、$s_RemoteFile共に現在のディレクトリからの相対パスの部分修飾名、あるいは完全修飾名です。

 

関連
